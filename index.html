LOGO! 0BA6 Program Dumper

Reads the circuit-program memory off a Siemens LOGO! 0BA6 over the serial PG protocol, in your browser (Chrome/Edge, desktop or Android). Two transports: Web Serial (the OS driver configures the cable — most reliable) and WebUSB (this tool configures the chip directly — needed where Web Serial doesn't list the cable). Nothing is uploaded — it all runs locally.

Before you start — plug the USB cable in, and put the LOGO! in STOP (ESC → Stop on the device; step 2 also does it). On Android use an OTG adapter and Chrome. Follow the numbered steps — the glowing button is the recommended next one. The log at the bottom records every byte; use Copy log to send it to me.

Do these in order. Steps 3 and 5 write to the PLC and require a second click to confirm.

not connected

Step 3 is only needed if the program is password-protected (the tool detects it): it recovers the password the 0BA6 stores in cleartext and lowers protection so step 4 can read; step 5 restores protection. No password → skip to step 4.

      Debugging aids. On a 0BA6.ES10 the program lives at 0x00FF0EE8 under the 0x00FF____ page; reads use single-byte Read Byte because Read Block is rejected on this hardware. Read Byte returns 0x00 for protected/unmapped memory (it does not fault), so an all-zero region means "protected or empty," not "read failed."
